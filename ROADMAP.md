# Island Life v4 Roadmap

## Phase 1 - Stabilize the Web Game

- Put the project in GitHub and use small commits for every feature or fix.
- Add automated checks for the game engine: action limits, finance calculations, annual settlement, saves, events, death, and generation transfer.
- Add a manual playtest checklist for new game, save/load, 10-focus year loop, event choices, business, property, family, and death/legacy flow.
- Separate long data tables from UI where possible so new island content can be added safely.

## Phase 2 - Improve Playability

- Add onboarding/tooltips for first-time players.
- Add clearer feedback after every action: money change, stat change, skill change, unlocked path.
- Balance early-game income, expenses, stress, and health so difficulty feels fair.
- Add more event variety and island-specific storylines.
- Add settings for sound, reduced motion, text size, and save management.

## Phase 3 - Mobile Readiness

- Make the layout feel native on small phones and tablets.
- Add touch-friendly controls and test every screen at mobile sizes.
- Add offline support and stronger save/export/import handling.
- Wrap the web game with Capacitor for Android and iOS builds.

## Phase 4 - Store Release

- Create app icons, splash screens, screenshots, trailer media, privacy policy, and support page.
- Confirm content rating and monetization strategy.
- Test on real Android and iPhone devices.
- Publish a closed test before charging for the game.
